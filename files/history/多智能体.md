# 一、方法框架

## 题目定位

在固定 token / 轮数 / 时间预算下，提出一种选择性通信策略，让多 agent 只在必要时交流、只传必要信息、只审关键分歧。

## 方法名备选

我建议先用这个工作名：

**SPARC**  
Selective Passing and Audited Routing under Communication budget

## 核心思路

方法分三段。

### Stage A：独立解题，先不通信

每个 agent 先独立输出：

- 候选答案
- 极短草稿 reasoning
- 自身置信度
- 一个“最不确定点”或“关键证据点”

这样做的目的是把简单题挡在通信前面，避免所有样本都进入全量 debate。近期 SID 就是沿着这个方向，用模型自信号驱动 debate 早退和内容压缩。

### Stage B：触发式通信，而不是默认全聊

只有满足以下条件之一时才进入通信：

- 多个 agent 初始答案不一致
- 平均置信度低
- verifier 认为当前答案不稳
- 轻量 difficulty estimator 认为题目难

这一步和 AAAI 2026 的 auction-based communication 的精神一致：通信应当是资源理性的，而不是 free-for-all。

### Stage C：选择性传递 + 局部审计

不传完整 CoT，只传压缩后的关键信息。通信内容分三档：

- 结论级：答案 + 置信度
- 分歧级：我与别人不同的关键一步
- 证据级：支撑我答案的最关键证据

最后也不做简单 majority vote，而是让 auditor 只审关键分歧点。这和 AgentAuditor 的核心思想一致：把全局裁决变成 reasoning tree 上 critical divergence points 的局部比较。

## 一个最小可跑版本

最稳的 first version 是：

- 2–3 个同模型 agent 独立解题
- 若答案一致且置信度高，直接 early exit
- 若不一致，只交换“关键分歧点”
- 由 1 个 auditor 做局部裁决
- 只在最难样本上进入第二轮通信

这样你们已经不是在做普通 debate，而是在做：  
independent solve → disagreement-triggered communication → local auditing

# 二、baseline 表

下面这张表可以直接拿去当实验设计清单。

| 类别 | Baseline | 作用 | 是否必须 |
|---|---|---|---|
| 单体下界 | Single-agent CoT | 单模型标准推理下界 | 必须 |
| 单体增强 | Self-consistency | 不通信情况下的多样采样对照 | 必须 |
| 经典多 agent | Vanilla Multi-Agent Debate | 最常见 debate 对照 | 必须 |
| 无通信聚合 | Majority Vote | 检验通信是否真有增益 | 必须 |
| 角色分工 | Solver-Critic / Proposer-Verifier | 常见双角色框架 | 建议 |
| 预算感知 | Auction-based communication | AAAI 2026 直接相关 baseline | 必须 |
| 自信号压缩 | SID | 早退 + 压缩 debate 内容 | 必须 |
| 局部审计 | AgentAuditor | 不靠 majority vote 的新基线 | 必须 |
| 去共识化 | Free-MAD | 不依赖多轮达成共识的对照 | 建议 |

这些 baseline 的依据分别是：  
AAAI 2026 的 auction-based communication 直接讨论预算下的语言 agent 交互；SID 用模型级 confidence 和 token-level semantic focus 做 debate 早退与内容压缩；AgentAuditor 用 reasoning tree 的关键分歧点替代 majority vote；Free-MAD 则明确批评多轮共识带来的 token 开销与扩展性问题。

# 三、数据集

这个题不适合一上来做太开放的长任务。  
最好选**能明确体现“通信是否必要”**的 benchmark。

## 推荐主组合

1. **GSM8K**

   适合看简单到中等难度数学题上，通信是否经常是浪费。

2. **MATH500**

   适合看难题上多 agent 是否真正值得通信。

3. **StrategyQA**

   适合看结论冲突和局部证据交流。

4. **HotpotQA**

   适合看“多跳证据是否值得交换”。

这四个组合已经够一篇方法论文的最小版，因为它们覆盖：

- 算术推理
- 高难数学
- 常识逻辑
- 多跳证据整合

多 agent / debate /审计类工作通常也会在 reasoning benchmark 上验证性能与 token 成本。SID 明确报告在多个 challenging benchmarks 上同时比较效果与 token 消耗；auction-based communication 也在 reasoning benchmarks 上评估预算与性能权衡。

## 可选扩展

- MuSiQue
- Date Understanding
- ProntoQA 小子集
- HumanEval 小子集

# 五、ablation 设计

这题最关键的 ablation 不是换 backbone，而是拆通信机制。

## Ablation 1：触发机制

比较：

- always communicate
- disagreement-triggered
- confidence-triggered
- verifier-triggered
- hybrid trigger

目的：证明“何时说话”本身就是贡献。

## Ablation 2：通信内容

比较：

- full CoT exchange
- answer only
- answer + confidence
- disagreement step only
- critical evidence only
- adaptive content

目的：证明“传什么”比“传更多”更重要。

## Ablation 3：裁决方式

比较：

- majority vote
- single judge
- final-round vote
- local auditing
- disagreement-point auditing

目的：证明局部审计优于全局投票。AgentAuditor 的存在正好给你这个方向一个强参照。

## Ablation 4：拓扑

比较：

- all-to-all
- star / controller
- pairwise challenger
- selective routing

目的：证明不是所有 agent 都该互相通信。

## Ablation 5：agent 数量

比较：

- 1
- 2
- 3
- 4

目的：证明收益与数量、预算、策略三者共同相关，而不是线性增加。

## Ablation 6：同质 vs 异质 agent

比较：

- 同一 7B 多采样
- 7B + 14B
- generalist + verifier

目的：看看收益来自“多样性”还是“角色功能”。

## Ablation 7：是否需要第二轮通信

比较：

- 最多 1 轮
- 最多 2 轮
- 最多 3 轮

Free-MAD 之所以有意义，就是因为它直接指出多轮共识会显著增加 token 开销且限制扩展性。这个 ablation 可以直接回应那一点。

# 七、学生可直接开做的最小路线

## 第 1 周

复现：

- Single-agent CoT
- Majority Vote
- Vanilla MAD

先在 GSM8K 和 StrategyQA 上通 pipeline。

## 第 2 周

加入：

- disagreement trigger
- answer-only / evidence-only 压缩通信

先看 token 是否明显下降。

## 第 3 周

加入：

- auditor
- local conflict checking

补 MATH500 和 HotpotQA。

## 第 4 周

对比：

- Auction-based
- SID
- AgentAuditor

然后出主表和 ablation。

# 八、这篇论文最该怎么写贡献

不要写成“我们研究多 agent 什么时候有帮助”。  
更应该写成：

- 提出一个预算感知的选择性通信框架，使多 agent 仅在必要时通信。
- 提出一种关键分歧点局部审计机制，把全局 debate 转化为局部验证。
- 在多个 reasoning benchmark 上证明，在相同预算下优于 vanilla MAD、auction-based communication、SID 和 AgentAuditor。
- 揭示多 agent 的主要收益来自高质量分歧解析与局部核查，而不是盲目增加 agent 数和轮数。这个结论与当前对多 agent debate 的最新反思是一致的。
