基于你现在这条 **SPARC / selective communication + local auditing** 的主线，我建议不要“见到多智能体就都读”，而是围绕四个问题来读：**无通信下界有多强、debate 到底有没有独立价值、何时该通信、最后该怎么裁决**。你文档里已经把 baseline 和数据集底盘定得比较对了：Single-agent CoT、Self-consistency、Majority Vote、Vanilla MAD、DALA、SID、AgentAuditor，以及 GSM8K / MATH500 / StrategyQA / HotpotQA 这组开发集，都是合理起点；这些开发集的本地目录分别在 [`../datasets/gsm8k/`](../datasets/gsm8k/)、[`../datasets/math500/`](../datasets/math500/)、[`../datasets/strategyqa/`](../datasets/strategyqa/)、[`../datasets/hotpotqa/`](../datasets/hotpotqa/)，论文总索引见 [`../references/README.md`](../references/README.md)。我会在这个基础上再补 **Debate-or-Vote、iMAD、MMLU-Pro、以及一个 topology/coordination benchmark**，而不是重起炉灶。 ([开放评审](https://openreview.net/forum?id=iUjGNJzrF1))

## 一、建议优先精读的论文

### 第一层：必须先读，而且基本都要复现

1. [**Chain-of-Thought Prompting Elicits Reasoning in Large Language Models**](../references/NeurIPS-2022-chain-of-thought-prompting-elicits-reasoning-in-large-language-models-Paper-Conference.pdf)
   这是你所有实验的最底层单体基线；没有 CoT，下游所有“多 agent 是否值得”都没有参照。([NeurIPS Proceedings](https://proceedings.neurips.cc/paper/2022/hash/9d5609613524ecf4f15af0f7b31abca4-Abstract-Conference.html?utm_source=chatgpt.com))
2. [**Self-Consistency Improves Chain of Thought Reasoning in Language Models**](../references/Self-Consistency%20Improves%20Chain%20of%20Thought%20Reasoning%20in%20Language%20Models.pdf)
   这是最关键的“无通信多样采样”强基线。你的方法后面要证明的，不是“多样性有用”，而是“在相同预算下，结构化通信比单体多采样更值”。([arXiv](https://arxiv.org/abs/2203.11171?utm_source=chatgpt.com))
3. [**Improving Factuality and Reasoning in Language Models through Multiagent Debate**](../references/Improving%20Factuality%20and%20Reasoning%20in%20Language.pdf)
   这是经典的 canonical MAD 论文。你后面做 Vanilla MAD、两轮 debate、最终投票，基本都要从这类最标准的协议开始搭。([arXiv](https://arxiv.org/abs/2305.14325?utm_source=chatgpt.com))
4. [**Debate or Vote: Which Yields Better Decisions in Multi-Agent Large Language Models?**](../references/Debate%20or%20Vote%20Which%20Yields%20Better%20Decisions%20in%20Multi-Agent%20Large%20Language%20Models.pdf)
   这篇是你现在最该严肃对待的“反方”。它把 MAD 拆成 majority voting 和 inter-agent debate 两部分，结论是多数提升主要来自 voting，而 debate 本身往往不提高期望正确率；官方还放出了代码。你后面所有实验，都要围绕“我到底比 vote 多了什么真实价值”来设计。([开放评审](https://openreview.net/forum?id=iUjGNJzrF1))
5. [**Auditing Multi-Agent LLM Reasoning Trees Outperforms Majority Vote and LLM-as-Judge**](../references/Auditing%20Multi-Agent%20LLM%20Reasoning%20Trees%20Outperforms%20Majority%20Vote%20and%20LLM-as-Judge.pdf)
   这篇基本就是你 Stage C / local auditing 最直接的对照：它把聚合从多数投票改成 reasoning tree 上关键分歧点的局部审计，并报告了相对 majority vote 和 LLM-as-Judge 的增益。你的“关键分歧局部核查”一定要读这篇。([arXiv](https://arxiv.org/abs/2602.09341))

### 第二层：与你的方法最像，建议精读并做 paper-faithful 复现

1. [**Cost-Effective Communication: An Auction-based Method for Language Agent Interaction (DALA)**](../references/Cost-Effective%20Communication%20An%20Auction-based%20Method%20for%20Language%20Agent%20Interaction.pdf)
   这篇是预算感知通信最直接的强 baseline。它把“说话权”建模成稀缺资源，核心思想是让 agent 为发言机会竞价，从而学会“strategic silence”。如果你要写预算感知 selective communication，这篇必须进主对照。([arXiv](https://arxiv.org/abs/2511.13193))
2. [**SID: Multi-LLM Debate Driven by Self Signals**](../references/SID%20Multi-LLM%20Debate%20Driven%20by%20Self%20Signals.pdf)
   这篇和你的 early exit / 内容压缩非常贴。它用 model-level confidence 和 token-level semantic focus 做 debate 早退与压缩；OpenReview 页面显示它是 ICLR 2026 的 withdrawn submission，所以适合当**方法灵感和重要 baseline**，但写论文时别把它当“已定论的 accepted SOTA”。([开放评审](https://openreview.net/forum?id=a2DVm6PMl0))
3. [**iMAD: Intelligent Multi-Agent Debate for Efficient and Accurate LLM Inference**](../references/iMAD%20Intelligent%20Multi-Agent%20Debate%20for%20Efficient%20and%20Accurate%20LLM%20Inference.pdf)
   这篇直接回答“什么时候值得触发 debate”。它先抽取单体回答里的 hesitation/self-critique 特征，再用轻量分类器决定要不要进入 MAD；论文在 arXiv 页面标注为 AAAI 2026 Oral。你如果想把 trigger 层做强，这篇值得认真读。([arXiv](https://arxiv.org/abs/2511.11306))
4. [**Free-MAD: Consensus-Free Multi-Agent Debate**](../references/Free-MAD%20Consensus-Free%20Multi-Agent%20Debate.pdf)
   这篇很适合拿来做“少轮数 / 非共识”对照：它强调 single-round、anti-conformity、trajectory-level scoring，而不是传统“多轮聊到共识再最后投票”。不过它同样是 ICLR 2026 的 withdrawn submission，适合作为思想与对照，不建议当论文唯一强证据。([开放评审](https://openreview.net/forum?id=46jbtZZWen))

### 第三层：帮助你写“为什么有效/为什么无效”的理论与反思

1. [**Multi-LLM Debate: Framework, Principles, and Interventions**](../references/Multi-LLM%20Debate%20Framework%20Principles%20and%20Interventions.pdf)
   这篇给的是理论框架，尤其重要的是它指出：当 agent 能力类似、响应相似时，debate 很容易静态收敛到多数意见；如果多数本身共享误解，debate 只会把错误共识放大。你写机制分析、错误分类时非常有用。([开放评审](https://openreview.net/forum?id=sy7eSEXdPC))
2. [**Multi-LLM-Agents Debate – Performance, Efficiency, and Scaling Challenges**](../references/Multi-LLM-Agents%20Debate%20Performance%20Efficiency%20and%20Scaling%20Challenges.pdf)
   这是一篇 ICLR 2025 blogpost track 的系统性评估，结论偏保守：现有 MAD 在多个 benchmark 上并不能稳定超过更简单的单体策略。它不是你的主 baseline，但很适合拿来提醒自己不要把“更多轮数/更多 agent”当自然正确方向。([开放评审](https://openreview.net/forum?id=Wv0J0bEly5))

## 二、建议补充读的 benchmark / 数据集论文

1. [**MMLU-Pro**](../datasets/mmlu-pro/)
   如果你要一个更像“主表硬 benchmark”的多选题集，我更推荐 MMLU-Pro 而不是原版 MMLU：它更偏 reasoning、把选项从 4 个扩成 10 个，并且对 prompt 更稳定。本地数据在 [`../datasets/mmlu-pro/`](../datasets/mmlu-pro/)，本地论文在 [`../references/MMLU-Pro A More Robust and Challenging Multi-Task Language Understanding Benchmark.pdf`](../references/MMLU-Pro%20A%20More%20Robust%20and%20Challenging%20Multi-Task%20Language%20Understanding%20Benchmark.pdf)。([arXiv](https://arxiv.org/abs/2406.01574))
2. [**HotpotQA**](../datasets/hotpotqa/)
   你做局部审计、证据级通信时，HotpotQA 特别适合，因为它有强监督 supporting facts，而且官方站点直接提供了 evaluation script。本地数据在 [`../datasets/hotpotqa/`](../datasets/hotpotqa/)。([HotpotQA](https://hotpotqa.github.io/))
3. [**GPQA**](../datasets/gpqa/)
   这是一个高难度 bonus set：448 道由领域专家写的 biology/physics/chemistry 多选题，设计目标就是“Google-proof”。规模不大，但很适合后期拿来做困难样本验证。本地目录在 [`../datasets/gpqa/`](../datasets/gpqa/)，本地论文在 [`../references/GPQA A Graduate-Level Google-Proof Q and A Benchmark.pdf`](../references/GPQA%20A%20Graduate-Level%20Google-Proof%20Q%20and%20A%20Benchmark.pdf)。([arXiv](https://arxiv.org/abs/2311.12022?utm_source=chatgpt.com))
4. [**HumanEval**](../datasets/humaneval/)
   如果你后面要把方法扩到代码任务，HumanEval 仍然是最自然的 pass@k / functional correctness 基准。本地数据在 [`../datasets/humaneval/`](../datasets/humaneval/)，本地论文在 [`../references/Evaluating Large Language Models Trained on Code.pdf`](../references/Evaluating%20Large%20Language%20Models%20Trained%20on%20Code.pdf)。([arXiv](https://arxiv.org/abs/2107.03374?utm_source=chatgpt.com))
5. [**AgentsNet**](../datasets/agentsnet/)
   如果你最后想证明“不是所有拓扑都一样”，AgentsNet 很适合作为 topology / coordination 的补充 benchmark；它关注的是 agent 在网络拓扑下自组织、协作和有效通信的能力，而不只是普通问答分数。本地数据在 [`../datasets/agentsnet/`](../datasets/agentsnet/)，本地论文在 [`../references/AgentsNet Coordination and Collaborative Reasoning in Multi-Agent LLMs.pdf`](../references/AgentsNet%20Coordination%20and%20Collaborative%20Reasoning%20in%20Multi-Agent%20LLMs.pdf)。([开放评审](https://openreview.net/forum?id=gsSIH0mZ0Y))
6. **GSM-Symbolic / GSM1k（作为 GSM8K 的警示文献）**
   我不会把 GSM8K 当你论文 headline benchmark 的唯一支柱。GSM-Symbolic 显示只改数值、加无关子句都可能让模型性能明显波动；GSM1k 的论文也指出部分模型在 GSM8K 上可能存在一定过拟合或污染迹象。更稳的做法是：**GSM8K 做开发集，MMLU-Pro/HotpotQA/更难数学集做主表。** 本地资源见 [`../datasets/gsm-symbolic/`](../datasets/gsm-symbolic/)、[`../datasets/gsm1k/`](../datasets/gsm1k/) 与 [`../references/A Careful Examination of Large Language Model Performance on Grade School Arithmetic.pdf`](../references/A%20Careful%20Examination%20of%20Large%20Language%20Model%20Performance%20on%20Grade%20School%20Arithmetic.pdf)。([arXiv](https://arxiv.org/abs/2410.05229))

## 三、哪些实验是“必须复现”的

### A. 第一批：一周内必须跑通

1. **Single-agent CoT / Self-consistency / Majority Vote 的 equal-budget 对照**
   这是你所有工作最重要的下界和上界。要求同 backbone、同 prompt 风格、同样的总调用预算，至少先在 GSM8K、StrategyQA、HotpotQA 小规模 dev 上跑通。没有这一步，后面所有通信方法都站不住。([NeurIPS Proceedings](https://proceedings.neurips.cc/paper/2022/hash/9d5609613524ecf4f15af0f7b31abca4-Abstract-Conference.html?utm_source=chatgpt.com))
2. **Vanilla MAD 复现**
   最小设置建议是 2–3 个同模型 agent、1–2 轮 all-to-all debate、最后 majority vote。你需要的不是最好成绩，而是一条最干净的“传统 debate”基线。([arXiv](https://arxiv.org/abs/2305.14325?utm_source=chatgpt.com))
3. **Debate vs Vote 受控实验**
   这组实验优先级极高。核心不是复现整篇论文所有细节，而是把你的环境下的同一批候选答案分别送去 “直接 vote” 和 “先 debate 再 vote”，看 debate 到底贡献了多少。因为这正是你论文最容易被问到的问题。([开放评审](https://openreview.net/forum?id=iUjGNJzrF1))

### B. 第二批：两周内做完机制复现

1. **Trigger / Early-exit 实验**
   至少做 4 个版本：always communicate、disagreement-triggered、confidence-triggered、hybrid trigger。这个实验同时覆盖了 iMAD 的“何时触发”和 SID 的“高置信度早退”两条线。([arXiv](https://arxiv.org/abs/2511.11306))
2. **Communication content ablation**
   你要明确比较：full CoT、answer only、answer + confidence、disagreement step only、critical evidence only。SID 的压缩逻辑和你自己的 selective passing 都需要靠这张表说话。([开放评审](https://openreview.net/forum?id=a2DVm6PMl0))
3. **Aggregation / Auditing ablation**
   至少比较：majority vote、single judge、final-round vote、local auditing。这个实验直接对应 AgentAuditor，也是你论文最能体现“不是只会少说话，而是更会裁决”的地方。([arXiv](https://arxiv.org/abs/2602.09341))

### C. 第三批：有资源就做，价值很高

1. **DALA-lite，再到 full DALA**
   我不建议你一开始就硬复现 DALA 的完整训练版。更合理的顺序是：先做一个 **DALA-lite**，也就是“固定发言预算 + 按 value density 选少量消息”的 paper-faithful 近似；如果这一步证明预算理性通信确实有价值，再决定是否投入时间做完整拍卖/学习版。DALA 本身的正式论文把通信建模成稀缺资源，且公开结果声称在多项 benchmark 上兼顾准确率和成本，但完整实现成本明显高于普通 prompt-level baseline。([arXiv](https://arxiv.org/abs/2511.13193))
2. **SID-lite 和 Free-MAD-lite**
   SID-lite 只做“置信度早退 + 压缩消息”，Free-MAD-lite 只做“single-round + anti-conformity prompt + 非最终轮投票”。这两者都比 full reproduction 更适合前期机制验证。([开放评审](https://openreview.net/forum?id=a2DVm6PMl0))
3. **Communication-necessary setting**
   我很建议你额外做一个“没有通信就很难”的设置，比如 split-context HotpotQA，或者一个小规模 AgentsNet。这样能避免审稿人说你只是“多采样+投票”。HotpotQA 自带 supporting facts，很适合做证据级交换；AgentsNet 则能直接测试拓扑和协调。([HotpotQA](https://hotpotqa.github.io/))

## 四、我会怎么排复现顺序

最合理的顺序是：

**第 1 阶段**：CoT → Self-consistency → Majority Vote → Vanilla MAD。
**第 2 阶段**：Debate-or-Vote 受控比较。
**第 3 阶段**：Trigger / early exit / compression（SID-lite、iMAD-lite）。
**第 4 阶段**：Aggregation / local auditing（AgentAuditor-style）。
**第 5 阶段**：Budget-aware routing（DALA-lite，再考虑 full DALA）。
**第 6 阶段**：single-round / anti-conformity（Free-MAD-lite）。
**第 7 阶段**：hard benchmark 与 communication-necessary setting。([开放评审](https://openreview.net/forum?id=iUjGNJzrF1))

## 五、最后给你的压缩版结论

真正**必须精读并优先复现**的，我会给这 8 篇：

CoT、Self-Consistency、Du et al. 的 Multiagent Debate、Debate-or-Vote、DALA、SID、iMAD、AgentAuditor。([NeurIPS Proceedings](https://proceedings.neurips.cc/paper/2022/hash/9d5609613524ecf4f15af0f7b31abca4-Abstract-Conference.html?utm_source=chatgpt.com))

真正**必须先跑出来**的 6 组实验，是：

1. equal-budget 的 CoT / Self-consistency / Majority Vote；
2. Vanilla MAD；
3. Debate vs Vote 受控对照；
4. trigger / early exit；
5. communication content ablation；
6. aggregation / local auditing。
   这 6 组跑扎实了，你就已经具备写出一版像样 workshop / conference draft 的骨架了。([开放评审](https://openreview.net/forum?id=iUjGNJzrF1))
